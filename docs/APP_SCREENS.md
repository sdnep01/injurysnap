[Login Screen]
Above figure is the login screen of the app. It supports the email and password authentication along with the Continue with Google option. User are displayed all the necessary information in this screen that are required to be authenticated.

[Register screen]
Above figure shows the register screen of the InjurySnap app. This page is accessible from the login screen and if a user is not registered to the app yet, then they can register themselves via this page. All of the required inputs are displayed in the page along with an action to accept the privacy policy and TOS of the app. Once everything is validated properly, user will be registered to the app.

[Home page, drawer]
Above figure is the main homepage where user lands in when they are authenticated. It shows all the main features of the app, including start scanning a new injury, check history, view nearest clinics and view a list of recent injuries. It also includes a drawer that gets opened from left side. Drawer includes a quick actions to redirect users to respective screens/feature and also includes an option to log out from the app with red color.

[Assessment Start Screen]
Above figure shows the starting point for a new injury assessment. This screen guides the user on how to take a proper photo of their injury providing crucial tips like ensuring good lighting and keeping the camera steady. It serves as a preparatory step before the actual camera interface is launched. Users can choose to take a new photo or upload one from their gallery, ensuring flexibility in how they provide the image for analysis.

[Camera Capture Screen]
Above figure represents the custom camera interface designed for InjurySnap. It features an overlay frame to help users position their injury correctly within the shot, ensuring the AI model receives optimal input. Essential tools like flash control and camera flip are easily accessible. A bottom sheet provides real-time tips for best results, reinforcing the importance of clarity and focus. This screen ensures the data capture is standardized for accurate assessment.

[Assessment Questions Screen]
Above figure displays the interactive questionnaire that follows the image capture. Here, users provide subjective context that the image alone might miss. Users select the specific body part from a visual grid, define the type of pain (e.g., sharp, throbbing), and use a slider to rate their pain intensity on a scale of 1 to 10. This multimodal data collection—combining visual and reported symptoms—allows for a more comprehensive and accurate injury evaluation.

[Analyzing Screen]
Above figure shows the feedback screen during the AI analysis process. It uses a progress bar and a checklist animation to keep the user informed and engaged while the system processes the image and data in the background. Steps like "Verifying image quality" and "Identifying injury markers" are checked off in real-time. This transparent UI reduces perceived wait time and assures the user that a thorough analysis is being conducted.

[Assessment Results Screen]
Above figure is the final output screen, presenting the AI's findings in a clear, actionable format. It displays the detected injury type (e.g., Mild Sprain) and a severity score (e.g., Moderate) visualized with a color-coded gauge. Crucially, it provides recommended next steps, such as seeking medical attention or applying RICE treatment, and an estimated recovery timeline. Immediate actions like "Find Nearby Clinics" are prominently placed to facilitate quick follow-up care.

[Nearby Clinics Screen]
Above figure illustrates the clinic locator feature, essential for users requiring professional medical help. The screen offers a dual-view interface: a map view for visualizing clinics geographically relative to the user's location, and a list view for detailed clinic information. Users can see key details like clinic names, ratings, distance, and opening hours. Quick action buttons allowing users to "Call" or "Book" an appointment directly from the app streamline the journey to recovery.

[Injury History Screen]
Above figure tracks the user's recovery journey over time. A timeline view displays past assessments, color-coded by severity, allowing users to monitor trends in their injury health. A summary dashboard at the top aggregates key statistics, such as total injuries and number of full recoveries. Filtering options allow users to quickly sort through past records based on severity, making it a digital health diary for their sports-related incidents.

[Notifications Screen]
Above figure serves as the central hub for all app alerts. It aggregates important updates such as completion of injury assessments, reminders for recovery exercises, and alerts about new clinics opening nearby. Unread notifications are clearly marked to grab attention. This screen plays a vital role in user retention and adherence to recovery plans by keeping the user engaged with timely and relevant information.

[Settings Screen]
Above figure allows users to personalize their app experience. Organized into clear sections like Account, Preferences, and Support, it gives users control over their data and interface. Key features include toggling push and email notifications, managing personal profile details, and accessing language settings. [Help & Support Screen]
Above figure provides users with assistance and resources. It features a search bar for finding specific help topics quickly. Quick access buttons lead to user guides and video tutorials for visual learners. A standardized FAQ section addresses common queries about assessment accuracy and data privacy.
